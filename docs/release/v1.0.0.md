# Crest Core v1.0.0 发布说明

发布日期：2026-06-30

Crest Core v1.0.0 是面向全新私有化生产环境的首个交付版本。该版本不承接旧版本原地升级链路，生产默认使用 OpenJDK 17、OceanBase Oracle 系统库、OceanBase Oracle 业务数据源，并提供 Kubernetes 多副本和 Docker Compose 单主机生产交付。

## 交付物

```text
ghcr.io/sevoniva/crest-core-service:v1.0.0
ghcr.io/sevoniva/crest-core-web:v1.0.0
crest-core-v1.0.0-linux-amd64-offline.tar.gz
crest-core-v1.0.0-linux-arm64-offline.tar.gz
installer/init-sql/ob-oracle/crest-core-schema.sql
deploy/kubernetes/
deploy/docker/
```

## 生产默认

- 系统库只支持 `CREST_DB_TYPE=ob-oracle`。
- 生产默认关闭 Flyway 和 Demo 数据，由 DBA 先执行首版初始化 SQL。
- Kubernetes 只启动 `crest` 前端和 `crest-service` 组合后端两个工作负载；组合后端以 2 副本承载 API、worker 和 scheduler。
- Docker Compose 只启动 `crest-core-web` 和 `crest-core-service` 两个服务；后端通过 `--scale crest-core-service=2` 运行两个副本。
- 外部共享 Redis Cluster 用于缓存、分布式锁、任务队列和 WebSocket 广播。
- 所有 Redis key、channel、stream 和 consumer group 必须使用同一个环境专属 hash tag，避免与其他系统冲突。
- 默认允许的业务数据源类型为 `obOracle,Excel,ExcelRemote,API`。
- 默认关闭 API 文档、SQLBot/AI、模板市场、字体自定义管理和背景资源库。

## 初始化

```bash
obclient --default-character-set=utf8mb4 \
  -h <obproxy-host> -P 2883 \
  -u '<user>@<tenant>#<cluster>' \
  -p'<password>' \
  < installer/init-sql/ob-oracle/crest-core-schema.sql
```

## 验收建议

- 使用 OpenJDK 17 构建后端。
- 执行前端生产构建和 lite 构建检查。
- 执行 `node scripts/generate-ob-oracle-init-schema.mjs --check`。
- 执行 `node scripts/verify-kubernetes-production.mjs deploy/kubernetes`。
- 执行 `node scripts/verify-docker-production.mjs deploy/docker`。
- 执行 `kubectl create --dry-run=client -f deploy/kubernetes -o name`。
- 对真实 Docker Compose env 执行 `node scripts/verify-docker-production.mjs deploy/docker --strict-config <env-file>`。
