# 平台管理

平台管理用于维护用户、组织、角色、菜单权限和业务资源权限。权限判断在后端执行，前端只负责展示菜单、按钮和授权界面。

## 管理范围

| 能力 | 说明 |
| --- | --- |
| 用户管理 | 新建、编辑、启停、删除、批量删除和批量导入用户 |
| 组织管理 | 维护多级组织树和用户默认组织 |
| 角色管理 | 维护角色、角色成员和角色授权 |
| 菜单权限 | 控制系统菜单和功能入口可见范围 |
| 资源权限 | 控制数据源、数据集、仪表盘和数据大屏可见范围 |
| 单点登录 | 支持外部身份登录后的用户查找、绑定或自动创建 |

## 权限模型

用户访问资源时，后端按以下维度计算可见范围：

- 当前用户是否为系统管理员；
- 用户创建的资源；
- 用户默认组织及下级组织资源；
- 直接授权给用户的资源；
- 授权给用户角色的资源；
- 授权给用户所属组织的资源。

前端菜单和按钮只作为交互提示，不作为最终权限依据。

## 数据对象

| 表 | 用途 |
| --- | --- |
| `core_iam_user` | 本地用户和外部身份绑定 |
| `core_iam_org` | 组织树 |
| `core_iam_role` | 角色 |
| `core_iam_user_org` | 用户与组织关系，包含默认组织 |
| `core_iam_user_role` | 用户与角色关系 |
| `core_iam_role_menu_permission` | 角色菜单权限 |
| `core_iam_resource_index` | 统一资源索引 |
| `core_iam_resource_permission` | 资源授权关系 |

资源索引用于统一描述数据源、数据集、仪表盘和数据大屏的资源类型、组织、创建人和删除状态。资源授权不替代资源本身的数据表，只作为权限过滤依据。

## 登录和用户上下文

本地登录成功后，后端生成包含用户 ID 和默认组织 ID 的 token。单点登录成功后，后端按外部唯一标识查找或创建用户，并使用同一套 token 与权限模型。

后端业务代码通过当前用户上下文获取用户 ID、默认组织和角色信息。接口不得信任前端传入的用户身份或组织范围。

## 管理入口

平台管理入口位于系统设置：

- 用户管理；
- 组织管理；
- 角色管理；
- 权限管理；
- 系统参数；
- 站点设置。

单点登录配置页、字体管理页等外围入口在 Crest Core 生产最小版中默认不展示；需要启用时必须同步打开对应 feature flag 并完成安全验收。

普通用户只能访问已授权资源。管理员可进入后台管理页面维护系统配置和授权关系。

## 数据导入

用户批量导入支持 `xlsx`、`xls` 和 `csv`。导入后用户必须具备：

- 账号；
- 姓名或展示名；
- 默认组织；
- 至少一个角色；
- 可用的初始密码或重置密码流程。

导入失败时应返回明确行号和原因，不暴露内部异常堆栈。

## 运维要求

- 首次安装后修改管理员密码；
- 定期检查管理员和高权限角色成员；
- 删除组织、角色和用户前确认资源授权影响；
- 启用单点登录后保留管理员应急登录路径；
- 权限异常排查时同时检查菜单权限和资源权限。

## 验收清单

- 非管理员无法访问后台管理入口；
- 普通用户只看到授权范围内的资源；
- 用户启停、删除和角色变更立即影响后续访问；
- 组织、角色、菜单权限和资源权限变更后，后端接口结果与前端展示一致；
- 单点登录用户能正确绑定本地用户或按配置自动创建；
- 用户导入、密码重置和初始密码提示可用。
