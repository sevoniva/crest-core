# 数据血缘

数据血缘用于回答三类问题：

- 这个字段从哪张表、哪个源字段来；
- 这个字段被哪些数据集、图表、仪表盘或数据大屏使用；
- 删除数据源或数据集之前，会影响哪些下游资源。

数据血缘是 Crest 的核心功能。它读取 Crest 已保存的元数据，整理成一张从数据源到展示资源的关系图。只要数据已经接入 Crest，并且被数据集或图表使用，就可以在血缘图里看到。

## 入口

登录后进入：

```text
数据血缘
```

新安装环境不附带示例业务库。生产环境直接在 `数据源` 中接入 OceanBase Oracle 业务库，再创建数据集和图表。血缘页面会读取这些已经保存的元数据，展示表、字段、数据集、图表和展示资源之间的关系。

## 使用方式

### 看一个数据源的血缘

1. 在左侧顶部第一个下拉框选择 `数据源`。
2. 在资源下拉框选择一个已经接入的数据源。
3. 页面会展示该数据源进入 Crest 后形成的链路。

链路从左到右展开：

```text
数据源 -> 物理表 -> 物理字段 -> 数据集字段 -> 数据集 -> 图表字段 -> 图表 -> 仪表盘/大屏
```

页面上方的统计卡片会显示当前图里的数据源、物理表、字段、数据集、图表和依赖关系数量。右侧列表分两块：资源列表用于确认当前图里有哪些节点；依赖列表用于按表格方式查看上游和下游。

### 按表和字段查血缘

字段级血缘需要先选择表，再选择字段：

1. 选择范围，例如 `数据源`。
2. 选择资源，例如某个已经接入的数据源。
3. 在 `选择表` 中选择物理表。
4. 在 `选择字段` 中选择该表下的字段。

选择字段后，图会收敛到这个字段相关的上下游，不再把整张资源图全部铺开。这个视图更适合排查字段来源、字段口径和影响范围。

### 按关键字查资源

搜索框支持字段名、资源名和类型名。输入关键字后点击 `查询`：

- 如果命中的是字段，页面会按字段血缘展开上下游；
- 如果命中的是数据集、图表或仪表盘，页面会展示与这些资源相连的依赖关系；
- 清空搜索后点击查询，会恢复当前资源的血缘。

### 看全局血缘

选择 `全局` 可以看整个 Crest 元数据范围内的资源关系。全局模式偏概览，主要用于确认资源分布和整体依赖，不默认展开所有字段细节，以免图过大影响浏览。

### 看数据集或仪表盘血缘

除了数据源，还可以切换到：

- `数据集`：从某个数据集反查上游表字段，并查看下游图表和仪表盘；
- `仪表盘/大屏`：从一个展示资源反查它使用的图表、数据集和源字段。

这两个入口适合在字段口径调整、数据集维护、看板下线前做影响确认。

## 图里的节点

| 节点 | 含义 |
| --- | --- |
| 数据源 | Crest 中配置的数据源，不包含数据源文件夹 |
| 物理表 | 被数据集引用过的源表或自定义 SQL 结果表 |
| 物理字段 | 物理表上的源字段 |
| 数据集字段 | Crest 数据集里的字段，包含原始字段和计算字段 |
| 数据集 | Crest 数据集资源 |
| 图表字段 | 图表里使用的字段位置，例如横轴、纵轴、标签、颜色、钻取、过滤条件 |
| 图表 | 仪表盘或数据大屏里的图表组件 |
| 仪表盘/大屏 | 最终展示资源 |

## 依赖关系

当前关系类型如下：

| 关系 | 含义 |
| --- | --- |
| `datasource_table` | 数据源包含物理表 |
| `table_table_field` | 物理表包含字段 |
| `table_field_dataset_field` | 物理字段映射为数据集字段 |
| `dataset_field_calc_field` | 计算字段引用其他数据集字段 |
| `dataset_field_dataset` | 数据集字段组成数据集 |
| `table_dataset` | 物理表参与构建数据集 |
| `dataset_chart` | 图表使用数据集 |
| `dataset_field_chart_field` | 图表字段引用数据集字段 |
| `chart_field_chart` | 图表字段属于图表 |
| `chart_dv` | 图表归属仪表盘或数据大屏 |
| `dataset_table_join_relation` | 数据集里的表关联 |
| `table_field_join` | 数据集关联键对应的字段关系 |

字段级查询就是沿这些关系向上、向下收集节点。

## 数据从哪里来

血缘不直接扫描业务库，也不去解析源库里的全部表。它只读取 Crest 自己保存下来的元数据：

| Crest 元数据 | 用途 |
| --- | --- |
| `core_datasource` | 数据源节点 |
| `core_dataset` | 数据集节点 |
| `core_dataset_table` | 数据集引用的物理表 |
| `core_dataset_field` | 物理字段、数据集字段和计算字段引用 |
| `core_chart_view` | 图表、图表字段和图表字段角色 |
| `core_visualization` | 仪表盘和数据大屏 |

这样做的好处是稳定、轻量，不会因为打开血缘页面就访问生产业务库。代价也很明确：没有接入 Crest、没有保存成数据集、没有被图表使用的库表，不会出现在血缘图里。

## 后端设计

后端入口在：

```text
core/core-backend/src/main/java/io/crest/relation
```

主要接口：

| 接口 | 说明 |
| --- | --- |
| `POST /relation/overview` | 全局资源关系 |
| `POST /relation/datasource/{id}` | 某个数据源的血缘 |
| `POST /relation/dataset/{id}` | 某个数据集的血缘 |
| `POST /relation/dv/{id}` | 某个仪表盘或数据大屏的血缘 |
| `POST /relation/resources/{type}` | 查询可选资源，`type` 支持 `datasource`、`dataset`、`dv`、`all` |

后端会先构建一个 `RelationContext`，把数据源、数据集、表、字段、图表和展示资源按 ID 分组。随后由 `GraphBuilder` 统一追加节点和边，并对节点、边做去重。

节点 ID 使用稳定规则生成。物理表和物理字段优先使用 `datasourceId + tableName + fieldName`，这样同一个数据源下被多个数据集重复使用的表字段不会在图上变成多个节点。数据集字段、图表字段、图表和仪表盘使用各自资源 ID 生成节点。

计算字段通过字段表达式里的字段 ID 建立关系。图表字段会读取横轴、纵轴、副轴、标签、提示、颜色、钻取、过滤条件等配置，尽量还原字段在图表里的用途。

## 前端设计

前端页面在：

```text
core/core-frontend/src/views/visualized/data/lineage/index.vue
```

图组件在：

```text
core/core-frontend/src/components/relation-chart/GraphView.vue
```

页面分为三块：

- 顶部筛选区：切换范围、资源、物理表、字段和关键字；
- 中间图谱区：用 ECharts graph 展示节点和方向；
- 右侧详情区：用表格列出资源和依赖，便于在节点较多时查找。

字段筛选在前端完成。后端返回当前资源图，前端根据选中的字段沿依赖边收集上下游，再渲染收敛后的图。这样切换字段时不需要反复请求后端。

大图做了降噪处理：节点或边较多时会关闭动画和拖拽、降低边线强调、隐藏部分字段标签，避免浏览器主线程被大图拖慢。

## 删除影响

血缘服务实现了原有的 `RelationApi`：

- 删除数据源前，检查是否还有下游数据集、图表或仪表盘；
- 删除数据集前，检查是否还有图表或仪表盘使用它。

这个检查用于阻止误删。它不是审批流，也不替代权限控制。

## 适用范围

Crest 当前覆盖平台内部建模后的字段级血缘，适用于数据源下线、数据集调整、看板变更和字段口径排查。血缘数据来自 Crest 已保存的资源关系，不会在页面访问时扫描业务库。

需要覆盖跨系统采集、完整 SQL 解析、版本历史、字段分级、质量状态或超大规模图谱检索时，应结合企业元数据治理平台统一设计。

## 回归检查

血缘相关代码发布前，至少使用测试业务库或预生产业务库回归以下场景。

- 打开 `数据血缘` 页面正常；
- 接入一个 OceanBase Oracle 业务数据源并保存至少一个数据集；
- `数据源`、`数据集`、`仪表盘/大屏` 三种范围都能加载；
- 表下拉框能列出当前资源涉及的物理表；
- 选表后能列出字段；
- 选字段后图谱收敛到字段上下游；
- 搜索字段名、数据集名、图表名都有结果；
- 清空搜索后恢复当前资源图；
- 删除被使用的数据源或数据集时仍能给出影响提醒；
- 仪表盘、数据大屏、系统设置页面打开正常；
- 浏览器控制台没有 chunk 404 或动态导入错误。
